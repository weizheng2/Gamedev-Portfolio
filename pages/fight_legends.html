Fight Legends

Dark Curry


  • Type: Professional Work
  • Links: Google Play, App Store
  • Role: Unity/Laravel FullStack Programmer
  • Studio: Dark Curry (Spain)
  • Time: November 2021 - Present

Fight Legends, also known as Dark Steel, is a mobile medieval RPG fighting game.
It is the project I was assigned to after finishing my internship with the studio and becoming a full-time programmer.
Below are some of my main contributions:

General

  • I'm in charge of making builds and submitting the game for different stores, ensuring compliance with their requirements.
  • Handle and process data and manage complex server systems (tournaments, rankings, and events).
  • Implement UI screen structure and functionality based on artist designs, taking multiple resolutions into account.
  • Run A/B testing processes where the game shows feature variations to test which performs better.
  • Ensure features and updates can be added without disrupting previous versions, allowing multiple versions to coexist across stores.

SDK and Plugins

  • Added advertisement plugins to show rewarded videos and interstitials. The probability to show is controlled via server.
  • Added analytics SDK like GameAnalytics and Firebase for receiving events required by the team.

Addressables

  • Implemented asynchronous asset loading using Unity Addressables System and a Google Play Addressables Plugin, overcoming Google Play's build size restrictions and enhancing performance by loading assets only when needed.
  • Previously used Amazon Web Services for storing the assets.

Local/Push Notifications

  • Local notifications with Unity Mobile Notifications for scheduled events, for instance, when the in-game store refreshes.
  • Push notifications with Firebase for instant actions processed on the server. One example is when a player receives a new chat message.
  • Also added an internal notifications interface for when the player is currently in game and with the screen focused.

Back System

  • The system allows users to close pop-ups, return to the previous screen, pause the game, etc., using an in-game back button or the device button.
  • The structure uses an event bus, and each screen can implement a different close event if needed.

IAPs, subscriptions, and bundles

  • For each store, I implemented validations for the receipts received in the server after a player buys any IAP.
  • Schedule cron jobs or tasks to update subscriptions data and process refunded, canceled or not given IAPs.
  • For the bundles, I also had to create conditions for showing and purchasing them, since they appear individually for each player and are not always available.

Tutorial

  • Implemented a comprehensive tutorial system, guiding players through core game mechanics, combat actions, and main interface screens.
  • In the combat part, I made a step-by-step guide where the battle only advances if the player performs the required action.
  • For other parts, the system blocks everything except the buttons or actions we want. The player is guided by an announcer and directional arrows.

Game Event

  • Created the first event in the game. It has a system where players can enter undefeated enemy rooms and continue where other players left.
  • Since multiple players can try to enter the same room, made sure to only let the first request in, avoiding any race condition.
  • Additionally, it contains a special shop where players can flip cards to get random rewards.

Chat system (mods, block, ban, etc.)

  • Created a chat system where players can either communicate in global channels or send private messages.
  • Global channels are managed in a server panel, making it easier to create and delete channels.
  • Players can also block/unblock others and if enabled to be a moderator, they have the ability to ban other players.

Game Shop

  • Implemented various shop sections where the player can buy different types of resources.
  • Items section, each player has its own catalog of items.
  • Chests section, players can get content with random possibilities.

Tournaments and rankings

  • Made a ranking system where players obtain points from playing a specific game mode. It rewards special icons for the chat.
  • Players can see the current week ranking updating in real time and also check last week's positions and the global all-time ranking.
  • In the server, each time the top ranking changes, it sends a push notification to the affected player.
  • Each week, a cron job is scheduled to process, qualify and reward rankings.

Multiple Characters

  • Developed a multi-character system, enabling players to create, switch, and manage multiple characters.
  • Since the game was designed for one character per user, the biggest challenge was to ensure the changes to the server and database did not affect the production game.

Referral System

  • Allows players to invite others and earn rewards.
  • Each code is generated with 'Hashids', to generate a unique, reversible, and short hash from the user ID.

Village Mode

  • Players can interact and upgrade buildings that provide different boosts.
  • Focused on decoupling the structure to facilitate the management of each building type.
